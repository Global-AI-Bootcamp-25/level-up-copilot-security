
## Objective

> Showcase what we can accomplish in security using all the latest features of GitHub Copilot.


# Official talk text

You already use GitHub Copilot to speed up development—but what if you could also write code according to your internal or industry security standards?

In this talk, we’ll show how to go beyond autocomplete, agent-mode, and leverage Copilot for security compliance and best practices. Whether your organization enforces strict standards or you need to align with frameworks like the OWASP Top 10, we’ll demonstrate how custom prompts, tailored instructions, and internal documentation can transform Copilot into a security ally without adding complexity.

In this session, our goal is to offer a practical, actionable approach on how to use the latest tools and capabilities of Copilot for security purposes.

# Automated Workflows

This repository uses [GitHub Agentic Workflows](https://github.github.com/gh-aw/) to automate routine maintenance and documentation tasks.

## Daily Documentation Updater

An agentic workflow that runs daily to keep documentation in sync with code changes.

- **Schedule**: Runs daily (via cron)
- **Engine**: GitHub Copilot
- **What it does**: Scans merged pull requests and commits from the last 24 hours, identifies new features or changes that should be documented, and opens a draft PR with documentation updates.
- **Output**: Draft pull requests labeled `documentation` and `automation`, assigned to `copilot` for review.

The workflow source is defined in [`.github/workflows/daily-doc-updater.md`](.github/workflows/daily-doc-updater.md) and compiled to [`.github/workflows/daily-doc-updater.lock.yml`](.github/workflows/daily-doc-updater.lock.yml).

## Agentics Maintenance

A maintenance workflow that automatically closes expired discussions, issues, and pull requests on a scheduled basis. This workflow is auto-generated by the `gh-aw` toolchain when any workflow uses the `expires` field.
